SMS VERIFICATION FOR LOGIN AND OWNER DASHBOARD ACCESS

Overview:
  Add Twilio SMS verification to improve security for platform login and owner dashboard access.

Acceptance Criteria:
  Login Verification:
    ✓ Send SMS code when user logs in
    ✓ Require code entry before issuing JWT token
    ✓ Verify user identity with a 6-digit OTP
    ✓ OTP expires after a short timeframe
    ✓ Allow resend with rate limiting
  Owner Dashboard Verification:
    ✓ Send SMS code before owner dashboard access is granted
    ✓ Require OTP validation before viewing owner-specific data
    ✓ Store verification attempts and expiry securely

Technical Notes:
  - Implement Twilio SMS service in backend
  - Add verification fields to personnel schema
  - Add login and owner dashboard verification flows in frontend
  - Use secure OTP generation and expiration

Priority: High
Dependencies: Auth module, owner dashboard, Twilio setup
